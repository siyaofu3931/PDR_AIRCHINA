# 国内测试问题分析与解决方案

## 1. 苹果与安卓航向镜像问题

### 现象
- 苹果手机：轨迹正常
- 安卓手机：航向镜像，往左拐弯时轨迹显示向右

### 原因
iOS 与 Android 的 `DeviceMotionEvent.rotationRate.alpha`（绕 Z 轴角速度）**符号约定相反**。当前引擎针对 iOS 使用 `gyro_alpha_sign = -1`，安卓需要取反。

### 解决方案（已实现）
- 前端在创建会话时发送 `platform: 'android' | 'ios'`
- 后端根据平台使用不同陀螺符号：iOS 用 -1，Android 用 +1

## 2. 国内地图与 OSM 的坐标偏差

### 现象
使用国内图源（如 GeoQ 智图）时，轨迹与 OSM/苹果地图相比存在固定偏移。

### 原因
- **WGS84**：国际标准，OSM、苹果地图使用
- **GCJ-02**：中国加密坐标系，高德、GeoQ、腾讯等国内图源使用
- 同一物理位置在两种坐标系下经纬度不同，偏差约几十至几百米

### 解决方案（已实现）
- 使用 GeoQ 等国内图源时，将 WGS84 坐标转换为 GCJ-02 再显示
- 中国境外（如 SFO）不转换，保持 WGS84

## 3. 国内网络延迟

### 现象
国内访问 Render 部署的应用时，网络较慢。

### 可选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **大陆服务器部署** | 延迟低、访问快 | 需 ICP 备案，流程复杂；需阿里云/腾讯云等 |
| **机场本地笔记本** | 无公网延迟，适合现场演示 | 需专人维护，非 7×24；需固定 IP 或内网穿透 |
| **香港/新加坡服务器** | 无需备案，国内访问相对较快 | 仍有一定跨境延迟 |
| **保持 Render + 瓦片代理** | 无需改部署，已实现 | 延迟取决于用户到 Render 的网络 |

### 建议
- **短期**：继续使用 Render + 瓦片代理，满足基本可用
- **正式商用**：考虑大陆服务器（阿里云/腾讯云）+ ICP 备案
- **现场演示**：可用笔记本本地部署 + 同一 WiFi 下访问
