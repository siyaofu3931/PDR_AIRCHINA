# 机场旅客定位测试 Playbook（国航内部预览版 v0.9）

本文档用于现场演示、联调、测试与问题定位，适用于当前 `v0.9` 页面结构与交互。

## 1. 版本范围

- 页面标题：`机场旅客定位测试(国航内部预览版v0.9)`
- 页面切换顺序：
  - 首页：`轨迹`
  - 第二页：`视频`
  - 第三页：`传感器`
- 关键 UI：
  - 轨迹卡片标题行右侧显示状态（`就绪 / 录制中`）
  - 地图上用户箭头可拖拽（用于位置/朝向调整）
  - 步数卡片仅保留一组 `VI 强度`（无重复）

## 2. 本地启动与对外访问

### 2.1 本地启动

在项目目录执行：

```bash
python3 -m http.server 8000
```

浏览器访问：

- `http://127.0.0.1:8000`

### 2.2 Tunnel（手机外网访问）

推荐 localtunnel：

```bash
npx localtunnel --port 8000
```

得到公开 URL 后，在手机 Safari 打开。

注意：

- iOS 传感器权限依赖 HTTPS / 安全上下文，务必使用 tunnel URL。
- 若页面看起来是旧版，先关闭旧标签页，再重新打开 URL 并强制刷新。

## 3. 标准操作流程（演示/测试）

1. 打开首页 `轨迹`。
2. 检查状态应为绿色 `就绪`（在轨迹卡片标题行右侧）。
3. 在地图上拖拽用户箭头：
   - 放到实际起点附近；
   - 按前进方向拉动（用于初始化朝向）。
4. 点击 `开始`。
5. 正常行走，观察：
   - `步数`递增；
   - `距离`递增；
   - 轨迹在地图上连续更新。
6. 点击 `停止` 后可下载：
   - `JSON`（轨迹与传感器日志）
   - `视频`（若本次录制成功启动相机）

## 4. 三页功能说明

## 4.1 轨迹页（首页）

- 显示：步数、距离、轨迹地图、提示文案。
- 交互：
  - 拖拽用户箭头更新位置与朝向；
  - `室内环境修正`开关；
  - `重置`（轨迹局部）与 `重置全部`（全局）。

## 4.2 视频页（第二页）

- 显示实时相机预览（录制期间）。
- 支持缩放控件（设备支持时）。
- 停止录制后可下载视频。

## 4.3 传感器页（第三页）

- 显示加速度、陀螺仪、姿态/磁力计曲线与数值。
- 主要用于调参与问题定位，不作为演示主界面。

## 5. 现场测试脚本（建议）

## 5.1 直线 20 米

- 目标：验证步频稳定、轨迹直线性、距离单调增长。
- 判定：
  - 步数持续增长；
  - 轨迹无明显抖动回跳；
  - 距离不出现 `NaN`。

## 5.2 长方形路径（四段+三次约 90° 转弯）

- 目标：验证转向响应（含 turn mode）与角点形态。
- 判定：
  - 转弯时轨迹能及时转向；
  - 角点不应大幅“拖尾”。

## 5.3 静止抗抖（站立 20 秒）

- 目标：验证误计步与虚假位移控制。
- 判定：
  - 步数不应持续增长；
  - 轨迹基本保持原地。

## 6. 常见问题排查

## 6.1 打开后仍是旧版本页面

- 原因：访问了旧 tunnel 或旧目录服务。
- 处理：
  - 确认标题包含 `v0.9`；
  - 重新获取 tunnel URL；
  - 手机端清缓存/强制刷新。

## 6.2 点击开始后步数不增长

- 检查：
  - 是否已授权 `运动与方向`权限；
  - 是否在 HTTPS（tunnel）下访问；
  - 传感器页曲线是否有变化。

## 6.3 转向慢或方向反了

- 检查：
  - 起步前是否拖拽箭头对齐朝向；
  - 是否处于强磁干扰区域；
  - 可通过 `室内环境修正`辅助约束。

## 6.4 无视频或无法下载视频

- 检查：
  - 浏览器相机权限；
  - 是否录制时长过短；
  - 设备是否支持 `MediaRecorder`。

## 7. 日志与回归

每次测试建议记录：

- 路径类型（直线/矩形/自由行走）
- 起止时间、设备型号、浏览器版本
- 是否使用室内修正
- 问题现象（步数、航向、轨迹、视频）
- 对应 JSON / 视频文件名

建议回归最小集合：

1. 直线 20 米
2. 长方形路径
3. 静止抗抖

保证每次改动后这三项均通过，再进入下一轮调参。
