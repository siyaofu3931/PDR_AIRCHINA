# PDR 室内导航增强方案 — 问题描述、系统设计与算法设计

**目标环境**: SFO 机场级室内（大空间、走廊、地标、磁干扰）  
**目标精度**: 米级定位；航向误差 &lt; 20°  
**传感器**: 陀螺仪、加速度计、磁力计（可选）；步数由算法计算  
**约束**: 通过用户交互（地标确认、初始朝向等）抑制发散，无额外硬件  

---

## 1. 问题描述

### 1.1 现状

- **现有 PDR**: 单页 WebApp，基于 `DeviceMotionEvent` / `DeviceOrientationEvent`，使用加速度幅值峰值检测步数、固定步长 0.7 m、直接用 orientation.alpha 作航向，在 2D 画布上绘制轨迹。
- **局限**:
  - 姿态固定：仅适合“手持看屏”；换姿势（口袋、握持）性能下降。
  - 无滤波：状态直接更新，易受噪声与抖动影响。
  - 无在线校准：无地标/距离校准，误差累积。
  - 固定步长：未根据步态（走/跑/楼梯）自适应。
  - 航向仅靠磁力计或设备朝向，机场内磁干扰大，易漂移。
  - 无环境约束：轨迹自由漂移，无法利用路网/走廊几何。
  - 无行为识别：电梯、扶梯、楼梯、跑步等未区分。
  - 无系统化测试：无 baseline、无 APE/RPE 等指标。
  - 无可视化：无 SFO 真实地图、无初始朝向对齐。
  - 无漂移预警与矫正引导。

### 1.2 需求摘要

| # | 需求 | 优先级 |
|---|------|--------|
| 1 | 手机姿态适应（手持/口袋等），算法随姿态调整 | P0 |
| 2 | 引入滤波（卡尔曼/互补/Madgwick 等）提升状态估计 | P0 |
| 3 | 在线校准：地标确认、已知距离校准步长系数 K | P0 |
| 4 | 自适应步长（如 Weinberg 等），K 可在线更新 | P0 |
| 5 | 环境约束：航向/位置与路网结合（Map Matching） | P0 |
| 6 | 行为识别：直梯/扶梯/楼梯/跑步/电梯等 | P1 |
| 7 | 测试框架：baseline JSON、APE/RPE 评估 | P1 |
| 8 | 标定流程：真实轨迹物理距离测量 | P1 |
| 9 | PoC：SFO OSM 地图 + 初始朝向 + 轨迹叠加 | P0 |
| 10 | 漂移检测与用户矫正提示 | P0 |
| 11 | 可视化规范：SVG/平面图、绿径、蓝点吸附、步数/距离 | P1 |

## 2. 系统设计

### 2.1 架构分层

```
┌─────────────────────────────────────────────────────────────────┐
│  UI 层: 初始朝向输入、地标确认、漂移提示、SFO 地图 + 轨迹        │
├─────────────────────────────────────────────────────────────────┤
│  行为与校准: 行为分类、地标校准、K 更新、漂移检测与提示           │
├─────────────────────────────────────────────────────────────────┤
│  状态估计: 航向融合(陀螺+磁)、步长估计(Weinberg)、位置预测      │
├─────────────────────────────────────────────────────────────────┤
│  路网约束 (Map Matching): 粒子滤波 / 投影到边 / 航向量化         │
├─────────────────────────────────────────────────────────────────┤
│  底层 PDR: 步态检测(姿态无关)、步周期 a_max/a_min、陀螺积分      │
├─────────────────────────────────────────────────────────────────┤
│  传感器: DeviceMotionEvent, DeviceOrientationEvent (IMU+Mag)    │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 数据流

1. **传感器** → 步态检测（幅值峰值 + 谷峰要求 + 最小周期）→ 步事件 + 本步 a_max, a_min。
2. **步事件** → 步长估计（Weinberg: K×(a_max−a_min)^0.25）→ 步长；航向（陀螺积分 + 磁力计互补）→ 航向角。
3. **步长 + 航向** → 位置预测 (x += L×sin(θ), y += L×cos(θ))。
4. **（可选）路网** → Map Matching：粒子在边上移动，按航向一致性加权，输出吸附到路网的位置。
5. **地标确认** → 重置粒子/位置到地标，并可选更新 K。
6. **漂移检测** → 触发“建议校准”提示或自动请求地标。

## 3. 算法设计

### 3.1 手机姿态适应（需求 1）

- **原则**: 使用 **幅值 |a|** 做步态检测，对手机朝向不敏感；不依赖单轴。
- **已有**: 当前已用 `sqrt(ax²+ay²+az²)`，已具备姿态无关基础。
- **增强**:
  - **低通**: 对幅值做一阶 IIR 低通（或 Butterworth），截止约 2–3 Hz，抑制高频抖动。
  - **峰值检测**: 保留“峰 + 谷”逻辑（需先有谷再计峰），最小步周期 400–500 ms，防抖。
  - **自适应阈值**: 用最近 N 步的幅值峰/谷做滑动统计，阈值 = baseline + α×std，减少固定阈值在手持/口袋下的误检。
  - **可选姿态分类**: 根据 beta/gamma 或重力方向将“手持/口袋/胸前”粗分类，仅用于调节阈值或步长系数（非必须）。

**实现要点**: 在现有 `stepDetect()` 中保留幅值 + 谷峰 + 最小周期，增加滑动窗口统计（最近 5–10 步的峰值）动态调整 minPeakHeight。

### 3.2 滤波与状态估计（需求 2）

- **航向**:
  - **陀螺积分**: heading(n) = heading(n-1) + ω_z × Δt（设备 z 轴角速度），短期准、长期漂移。
  - **磁力计**: 提供绝对航向，机场内易受干扰。
  - **互补滤波**:  
    `heading_fused = α × heading_gyro + (1−α) × heading_mag`  
    α 按磁可信度动态调整：磁场稳定（强度接近地磁、变化小）→ α 小（更信磁）；异常 → α≈0.95–0.98（几乎只用陀螺）。
  - **可选 Madgwick**: 若用四元数姿态（acc+gyro+mag），可用 Madgwick 滤波得到航向；与互补滤波二选一即可。

- **位置**: 仍为步进式更新；若引入路网，用粒子滤波在路网上做状态估计，见 3.5。

**实现要点**: 维护 `heading_gyro`（积分）、`heading_mag`（来自 orientation.alpha 或 atan2(my,mx)），计算磁场强度/稳定性，动态 α，输出 `heading_fused` 用于步进更新。

### 3.3 在线校准与地标（需求 3）

- **地标类型**: 用户可点击“我到达 [地标 A]”，系统记录当前 PDR 位置与地标真实坐标；从 A 走到 B 后点击“我到达 [地标 B]”。
- **校准内容**:
  - **步长系数 K**: 已知距离 d_AB，步数 n，当前平均步长 L_avg → K_new = d_AB / (n × L_avg) 或基于 Weinberg 反解 K。
  - **位置重置**: 到达 B 时将所有粒子或单点位置设到 B，消除累积误差。
- **交互**: UI 提供“设置起点”“我到达此地标”“输入已知距离”等，后端/前端保存 (地标 id, 真实坐标, 可选距离)。

**实现要点**: 在 WebApp 中增加“Landmark: 到达”按钮；在状态中保存 last_landmark_position 与 optional known_distance；在下一步长模型中用已知距离更新 K（见 3.4）。

### 3.4 自适应步长与 Weinberg（需求 4）

- **Weinberg**:  
  `step_length = K × (a_max - a_min)^0.25`  
  每步用本步周期内的幅值 a_max、a_min 计算当步步长；K 为全局校准系数。
- **K 的更新**:  
  当用户完成“从 A 到 B”且已知 d_AB、步数 n：  
  若当前平均步长估计为 L_avg（由过去 n 步的 Weinberg 估计平均），则  
  `K_new = d_AB / sum( (a_max_i - a_min_i)^0.25 )`  
  或简单用 `K_new = d_AB / (n × L_avg)` 再与当前 K 做指数平滑。
- **鲁棒性**:
  - 对 (a_max - a_min) 做上下限裁剪，避免异常步（如电梯内）拉偏 K。
  - 仅在使用“已知距离”的校准段更新 K，避免单次异常路径破坏 K。

**实现要点**: 每步检测到峰时，记录本步的 a_max、a_min（在步周期内用滑动窗口或步内缓冲）；步长 = K * (a_max - a_min)^0.25；校准时用 d_AB 与 n 更新 K。

### 3.5 环境约束与 Map Matching（需求 5）

- **数据**: 从 SFO OSM 或自采数据得到路网（走廊/通道）：节点 + 边，边带长度与方向。
- **思路**:
  - **航向量化**: 走廊多为直线或 90° 转弯；将 PDR 航向量化到最近边方向（如 0/90/180/270 或按边方向离散），减少小角度抖动。
  - **粒子滤波 Map Matching**:
    - 粒子只在边上，每粒子 (edge_id, offset)。
    - 每步：粒子沿边前进 step_length；到节点时按路网拓扑分叉，权重与 PDR 航向和边方向一致性相关。
    - 观测更新：PDR 航向与边方向一致 → 高权重；不一致 → 低权重。
    - 输出：粒子加权平均或最高权重点，作为“吸附到路网”的位置。
  - **地标**: 用户确认到达地标时，粒子重置到该地标所在边/节点。

- **SFO OSM**: 可从 OpenStreetMap 导出 SFO 区域 building/indoor 或 corridor；需解析为节点-边图。若 OSM 无足够室内细节，可用自绘平面图 + 手工路网 JSON。

**实现要点**: 先实现“航向量化到最近边”和“位置投影到最近边”的简化版（无粒子）；路网用 JSON 存储；后续再上粒子滤波。

### 3.6 行为识别（需求 6）

- **直梯/扶梯**: 无周期性步态、速度来自外部；检测“长时间无步数 + 加速度模式平缓/线性”或“恒定速度”，标记为 ELEVATOR/ESCALATOR，位移用时间×估计速度或用户输入。
- **跑步**: 步频高、步长短；用步周期 &lt; 400 ms 或 a_max−a_min 明显更大 区分 run/walk。
- **上下楼梯**: 步频较低、幅值模式不同；可用幅值包络或频谱区分 upstairs/downstairs。
- **电梯**: 无步数 + 垂直加速度/气压变化（若有气压计）；或简单用“无步数 + 持续时间 &gt; T”标记为 ELEVATOR_MOVE。

**实现要点**: 在步检测旁增加轻量状态机：last_step_ts、无步时长、步周期序列；输出 behavior_tag: 'walk'|'run'|'stairs_up'|'stairs_down'|'elevator'|'escalator'，并写入导出 JSON，便于后续做位移模型。

### 3.7 测试与 baseline（需求 7）

- **Baseline**: 指定路线录制“真值”轨迹 JSON（或从 EVO 格式的轨迹文件导入）：时间戳 + 位置 (x,y) + 可选航向。
- **比对**: 回放同一段 IMU/事件流，得到估计轨迹；与 baseline 对齐（时间对齐或最近点）。
  - **APE** (Absolute Position Error): 各时刻位置误差的范数统计（如 RMSE）。
  - **RPE** (Relative Position Error): 相对位姿误差（逐段相对位移的误差）。
- **实现**: 离线脚本或 WebApp 内“加载 baseline JSON + 当前轨迹”，计算 APE/RPE 并输出 log。

**实现要点**: 定义 baseline JSON schema（timestamps, positions[]）；在导出中增加“轨迹序列 + 时间戳”，便于与 baseline 对齐；提供简单 APE/RPE 计算脚本（如 Node 或 Python）。

### 3.8 标定与真实距离（需求 8）

- **目的**: 获得真实轨迹的物理距离，用于校准 K 和评估误差。
- **方法**: 用卷尺/激光测距在场地量取关键段长度；或沿固定路线多次用 PDR 取平均并用人造地标（已知距离）反推。记录“路线 id + 真实距离”到标定 JSON，供 K 更新和 APE 使用。

### 3.9 PoC：SFO 地图 + 初始朝向（需求 9）

- **地图**: 使用 Leaflet/Mapbox 或静态 SVG 加载 SFO 航站楼平面图；底图来自 OSM 或自绘。
- **初始朝向**: 用户点击“对齐地图”：先输入或选择当前朝向（如 0°/90°/180°/270° 或拖拽箭头），将 PDR 的 heading 零点与地图北/走廊对齐。
- **轨迹叠加**: 将 PDR 轨迹（或 Map Matching 后位置）转换到地图坐标系，绘制绿线 + 当前点蓝点；可选“吸附到路网”时蓝点在边上。

**实现要点**: 增加地图容器（Leaflet + OSM tile 或静态图）；存储 map_origin 与 heading_offset；将 (x,y,heading) 转为地图坐标后绘制。

### 3.10 漂移检测与矫正（需求 10）

- **检测**: 航向漂移：|heading_mag - heading_gyro| 持续偏大；位置漂移：与路网约束位置偏差持续增大，或粒子权重熵突增。
- **策略**: 当“估计漂移概率”超过阈值时，提示“建议进行地标校准”或“请确认当前所在位置”；用户确认地标后重置位置并可选更新 K。

**实现要点**: 计算 mag vs gyro 航向差、滑动窗口内方差；超过阈值则 setMessage 提示并可选弹出“是否到达地标？”。

### 3.11 可视化规范（需求 11）

- **底层**: SVG 或 Canvas 航站楼平面图；或 Leaflet 叠加层。
- **内容**: 绿色路径轨迹、路标节点、DR 实时位置蓝点（可吸附到路网）、已走路径、步数/距离实时更新。
- **数据**: 底层仍为 PDR 引擎（IMU → 步态 → 步长 + 航向 → 位置）；上层为路网约束与绘制。

## 5. 自测与测试 log

- **单元**: 步检测：给定合成加速度序列，检查步数与步周期；步长：给定 a_max/a_min，检查 Weinberg 输出。
- **集成**: 用录制 JSON 回放，对比轨迹与 baseline，输出 APE/RPE 到 tests/log/。
- **真机**: 在固定路线（已知距离）上走，检查步数、总距离、航向稳定性；记录“日期 + 设备 + 路线 + 步数 + 距离 + 备注”到测试 log。

建议在 `tests/log/` 下以日期或 run_id 命名保存每次测试的导出 JSON 与简短结论（通过/失败、APE 值等）。
